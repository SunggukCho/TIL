## 왜 DDD를 말하는가?

- Complexity: 제품의 복잡성을 해결
- Communication: 사람들간의 협업
- Similarity: 기존 프레임워크, 패턴과의 유사성
  - Redux 와 유사한 CQRS 패턴
    - Action vs Commands
    - Selector vs Query
    - Reducer에서 처리된 Event vs Domain Events
